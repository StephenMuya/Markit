# Compliance, Licensing, and Responsible Use

## Purpose
Define operating boundaries for lawful, sustainable commercialization of Markit intelligence outputs.

## Core Rules
1. Respect each source's terms of use and access policies.
2. Preserve source attribution for extracted events and claims.
3. Avoid publishing full-text article reproduction as a product substitute.
4. Prefer normalized facts/signals + links back to origin.
5. Maintain internal auditability of extracted outputs.

## Content Handling Guidelines
- Store canonical source URLs for every event.
- Keep extraction metadata for quality review.
- Flag low-confidence outputs for human verification.
- Correct or retract records when source evidence changes.

## Licensing-Sensitive Sources
Some data providers may impose strict usage rights for downstream commercial products.
- Validate rights before using provider data in paid offerings.
- Where redistribution is restricted, keep usage internal or provide only compliant derivative analytics.

## User-Facing Transparency
Markit should communicate:
- Intelligence is derived from third-party public/professional sources.
- Signals are assistive, not legal/financial advice.
- Users should validate material decisions independently.

## Security and Privacy Alignment
- Keep operational secrets in environment variables.
- Restrict access to production data and logs.
- Follow least-privilege principles for service/database access.
